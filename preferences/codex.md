# Codex Preferences

Use this file for durable instructions that should follow you across projects.

## Working Style

- Be direct and pragmatic.
- Prefer existing project conventions over new abstractions.
- Explain assumptions when they affect implementation choices.
- Verify changes with the narrowest meaningful test or check.

## Editing

- Keep changes scoped to the request.
- Avoid unrelated formatting churn.
- Do not revert user changes unless explicitly asked.

## Communication

- Lead with the outcome.
- Mention tests or checks run.
- Call out blockers plainly.
