# anna

Personal brain repo for Codex: skills, preferences, reusable context, and documentation.

## Layout

- `preferences/` - stable working preferences Codex should follow.
- `skills/` - custom Codex skills and skill drafts.
- `documentation/` - notes, references, project context, and reusable facts.

## Suggested Workflow

1. Put durable operating preferences in `preferences/codex.md`.
2. Keep evergreen documentation in `documentation/`.
3. Draft skills in `skills/` before installing or publishing them.
4. Commit changes when a note becomes worth preserving.
